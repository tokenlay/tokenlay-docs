# API Reference

Complete API documentation for Tokenlay.

## Overview

The Tokenlay API provides programmatic access to all features. This reference documents every class, method, and configuration option.

## Core API

### Tokenlay Class

The main entry point for using Tokenlay.

```typescript
import { Tokenlay } from 'tokenlay'

const tokenlay = new Tokenlay(options)
```

#### Constructor Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `apiKey` | `string` | - | Your API key |
| `endpoint` | `string` | `https://api.tokenlay.com` | API endpoint |
| `timeout` | `number` | `30000` | Request timeout in ms |

#### Methods

##### `process(data: ProcessOptions): Promise<Result>`

Process data through Tokenlay.

```typescript
const result = await tokenlay.process({
  input: 'your data here',
  options: {
    // processing options
  }
})
```

## Configuration

### Environment Variables

- `TOKENLAY_API_KEY` - Your API key
- `TOKENLAY_ENDPOINT` - Custom API endpoint
- `TOKENLAY_LOG_LEVEL` - Logging level (debug, info, warn, error)

## Error Handling

All API methods may throw errors. Always wrap calls in try-catch blocks:

```typescript
try {
  const result = await tokenlay.process(data)
} catch (error) {
  if (error.code === 'RATE_LIMIT') {
    // Handle rate limiting
  }
}
```

## Rate Limits

- 1000 requests per minute
- 10,000 requests per hour
- Contact support for higher limits
